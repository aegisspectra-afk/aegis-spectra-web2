# תוכנית שיפור חבילות מותאמות - רשימה מפורטת

## סקירה כללית
שיפור והרחבה של מערכת החבילות המותאמות, יצירת עמודים נפרדים לכל חבילה, ואופציות בחירה מתקדמות.

## 1. שיפור מבנה הנתונים של החבילות

### 1.1 יצירת טיפוסים (Types) מרכזיים
- [ ] יצירת קובץ `frontend/src/types/packages.ts` עם הגדרות:
  - `PackageType` - טיפוס חבילה (apartment, house, business, enterprise)
  - `PackageCategory` - קטגוריה (Residential, Commercial, Enterprise) - **לפילטרים ול-SEO**
  - `PackageFeature` - תכונות החבילה (מצלמות, NVR, AI, וכו')
  - `PackageSpecification` - מפרט טכני מפורט
  - `PackagePricing` - מחירים דינמיים (מינימום, מקסימום, טווח)
  - `PackageAddon` - תוספות זמינות לחבילה
  - **שדות נוספים בטיפוס:**
    - `slug` - SLUG ייחודי לחבילה (חלק מה-PackageType)
    - `id` - מזהה ייחודי (חלק מה-PackageType)
- [ ] שילוב JSON Schema Validation (Zod או Yup) - **לוודא עקביות במבנה הנתונים**

## סיכום
**סה"כ משימות: ~100 משימות** (כולל המלצות לשיפור)

**עדיפויות:**
1. **גבוהה**: סעיפים 1-4 (מבנה נתונים, תוכן, עמודים נפרדים, מחשבון)
2. **בינונית**: סעיפים 5-8 (עמוד ראשי, השוואה, אינטגרציה)
3. **נמוכה**: סעיפים 9-13 (SEO, עיצוב, תמונות, תיעוד)

**הערכת זמן:**
- סעיפים 1-4: ~5-7 שעות (כולל שיפורים)
- סעיפים 5-8: ~4-5 שעות (כולל שיפורים)
- סעיפים 9-13: ~2-3 שעות
- סעיף 14 (שדרוגים אסטרטגיים): ~6-8 שעות (עתידי)
- **סה"כ: ~11-15 שעות עבודה** (ללא סעיף 14)
